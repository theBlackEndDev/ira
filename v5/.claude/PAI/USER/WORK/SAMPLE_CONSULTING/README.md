# SAMPLE_CONSULTING

> 💼 SAMPLE TEMPLATE — Rename this directory (e.g. `Consulting/`, `Advisory/`) and populate it with your own work context via /interview.

This directory holds your consulting practice — the offerings you sell, the templates you reuse, and the proposals and statements of work currently in flight.

## What goes here

- **`Offerings/`** — descriptions of each consulting service you offer (e.g. assessments, advisory retainers, workshops). One file per offering, with scope, deliverables, pricing model, and ideal-fit customer.
- **`Templates/`** — reusable templates for proposals, SOWs, NDAs, project plans, kickoff decks. Anything you copy-paste-customize for each new engagement.
- **Proposals and SOWs in flight** — drop active proposal documents directly in this directory, named so your DA can identify the prospect (e.g. `Acme_Assessment_Proposal.pdf`).
- **Letters of recommendation, case studies, references** — anything that supports selling future engagements.

## Sample fields to populate

- **Practice name:** (interview — e.g. "Independent Security Advisory")
- **Primary offerings:** (interview — list 2–4 services)
- **Typical engagement size:** (interview — duration and dollar range)
- **Ideal customer profile:** (interview — company size, industry, role)
- **How you find new work:** (interview — referrals, inbound, network, marketplace)

## How to use

1. Rename this directory to match how you refer to your consulting practice in conversation.
2. Create `Offerings/` and add one file per service.
3. Create `Templates/` and drop in your reusable proposal/SOW/NDA templates.
4. Drop active proposal documents directly here as they come in.

## Privacy

Active proposal documents may contain prospect names, scope, and pricing. This directory is part of your private USER/ tree and is **not** included in PAI public releases.
